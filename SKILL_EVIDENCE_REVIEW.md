# Skill: Evidence Review

## Purpose

Review course content for definition clarity, evidence support, and
testable claims.

## Inputs

- changed module/docs/template/example files
- source references used in the change

## Outputs

- blocking issues
- non-blocking improvements
- pass/fail recommendation

## Constraints

- Do not approve abstract claims without definition and example.
- Do not approve vague quality language without criteria.

## Checklist

- Key concepts are defined before use.
- Each concept has a concrete example.
- Metaphors (if used) appear after definition and example.
- Learning objectives use measurable verbs.
- Claims match cited or demonstrated evidence.
- Exercises evaluate observable outcomes.

## Verification

- Cross-check with `EVAL_CHECKLIST.md`.
- Escalate unresolved blocking issues before completion.
