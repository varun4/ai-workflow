# Glossary

## Agent

A software system that can interpret instructions, use tools, and
produce work with partial autonomy.

## Workflow

A structured sequence of tasks, artifacts, decisions, and review steps
that turns intent into output.

## Skill

A reusable set of instructions for a recurring capability such as spec
writing, review, or evaluation.

## Spec

A structured definition of the task, scope, constraints, and success criteria.

## Context

The information package an agent needs to perform a task correctly.

## Context Architecture

The design of what context is included, excluded, persisted, and
refreshed across a workflow.

## Evaluation

The process used to verify whether an output is acceptable, complete, and safe.

## Failure Pattern

A recurring way workflows break, such as vague scope, missing context,
or unchecked assumptions.

## Approval Boundary

A point where human permission is required before the workflow can proceed.
