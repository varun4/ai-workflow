# Failure Patterns

This document should become a living catalog of recurring workflow failures.

## Initial Pattern List

- Vague spec, confident output
- Scope overreach
- Missing approval before risky actions
- Partial completion presented as complete
- Tool misuse caused by weak instructions
- Context drift across long sessions
- Review skipped because the output looked plausible

## Pattern Template

- Symptom
- Root cause
- Detection signal
- Prevention pattern
- Recovery action

## Teaching Goal

Learners should become good at recognizing failures as workflow design
problems rather than treating every miss as a model limitation.
