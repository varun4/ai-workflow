# Evaluation

Evaluation is the mechanism that decides whether output is acceptable.

## Evaluation Layers

- Correctness: did the work solve the stated problem?
- Completeness: is anything missing?
- Constraint compliance: were boundaries respected?
- Quality: is the result usable and maintainable?
- Safety: were risky actions avoided or approved?

## Evaluation Inputs

- Spec
- Acceptance criteria
- Checklists
- Tests or verification commands
- Human review when needed

## Teaching Points

- Evaluation should be designed before execution starts
- A good workflow separates generation from review
- "Looks good" is not a reliable evaluation standard
