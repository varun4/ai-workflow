# Skills

Skills capture repeatable instructions for tasks that appear often
enough to deserve standardization.

## When To Create A Skill

- A task is repeated across projects
- The task has a stable structure
- You want more consistent outputs
- A recurring failure pattern can be prevented with clearer instructions

## Skill Anatomy

- Purpose
- Inputs
- Outputs
- Constraints
- Checklist
- Verification

## Teaching Points

- Skills should stay narrow and reusable
- Skills are workflow assets, not personal prompt fragments
- A skill should reduce ambiguity, not add ceremony
