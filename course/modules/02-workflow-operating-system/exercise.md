# Module 02 Exercise

## Scenario

You lead a small team that wants to use AI agents in one recurring workflow.

Choose one:

- shipping a small software feature
- preparing a research synthesis
- producing a structured meeting summary
- handling a recurring operations task

## Task

Design a minimal workflow operating system for that workflow.

Your design must include:

1. A short description of the workflow
2. Three rules that would belong in `AGENTS.md`
3. A first entry for `PROGRESS.md`
4. A short spec with:

   - objective
   - scope
   - constraints
   - success criteria

5. One skill that would improve repeated execution
6. One evaluation checklist
7. Two ask-first approval boundaries

## Expected Output

A one-page workflow design that another person could use as the
starting point for a repeatable agent-assisted process.

## Evaluation Rubric

A strong submission should:

- define a clear workflow
- show explicit boundaries
- include at least one realistic approval gate
- make evaluation concrete
- avoid vague phrases like "improve" or "help with everything"

## Debrief Prompt

After completing the exercise, explain:

- which artifact was hardest to define
- where ambiguity still remains
- what could go wrong if the workflow ran without evaluation
