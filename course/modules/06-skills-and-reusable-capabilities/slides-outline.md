# Module 06 Slides Outline

## Slide 1: What A Workflow Skill Is

- plain-language definition
- why skills follow context architecture

## Slide 2: Why Skills Matter

- repeatability
- reliability
- reviewability
- lower rework

## Slide 3: When To Create A Skill

- task repeats across phases or sessions
- structure is stable
- quality drift is visible

## Slide 4: Skill Contract Anatomy

- purpose
- inputs
- outputs
- constraints
- checklist
- verification

## Slide 5: Skill Lifecycle

- identify repeatable task
- define skill contract
- invoke in workflow
- evaluate
- revise and version

## Slide 6: Failure Patterns

- broad skills
- missing verification
- stale skill logic
- no invocation trigger

## Slide 7: Coding Anchor Skill Example

- `SKILL_RBAC_CONTEXT_REFRESH.md`
- trigger points by phase
- expected output artifact

## Slide 8: Knowledge-Work Skill Example

- `SKILL_EVIDENCE_CONTEXT_REFRESH.md`
- source policy alignment
- expected output artifact

## Slide 9: Bad vs Good Skill Design

- catch-all prompt vs narrow capability
- implied quality vs explicit checklist

## Slide 10: Exercise Instructions

- design one skill per anchor
- define invocation and verification

## Slide 11: Debrief

- what changed after skill design?
- where does risk remain?
