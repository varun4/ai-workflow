# Module 06 Facilitator Notes

## Teaching Emphasis

- Teach skills as reusable capability contracts.
- Enforce one-skill-one-capability discipline.
- Require explicit invocation timing and verification.
- Use both anchors to show transferability.

## Core Message

A skill is useful only when it is narrow, testable, and tied to
observable workflow outputs.

## Common Student Mistakes

- writing broad skills that mix multiple capabilities
- omitting required inputs
- defining outputs as vague quality claims
- skipping stop conditions and escalation rules
- failing to define when the skill is invoked

## Facilitation Prompts

- "What repeated task does this skill standardize?"
- "What output proves this skill run happened?"
- "What should force this skill to stop and escalate?"
- "What event should trigger skill revision?"

## Suggested Timing

- 10 min: definition and rationale
- 15 min: anatomy and failure patterns
- 20 min: dual-anchor skill design exercise
- 10 min: debrief and revision planning

## Exit Check

Before closing, ask each learner to provide:

- one complete skill for the coding anchor
- one complete skill for the knowledge-work anchor
- one verification artifact they will require in real workflow
