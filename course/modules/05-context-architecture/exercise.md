# Module 05 Exercise

## Scenario

Design context architecture for both anchor workflows:

1. Coding: add role-based access checks to admin endpoints.
2. Knowledge work: prepare a vendor selection recommendation for
   leadership.

## Task

For each anchor, produce a context package with:

- required context sources
- excluded context sources
- source precedence rules
- context refresh triggers
- context owner per phase
- validation checks for context quality

## Required Output Format

For each anchor, submit:

1. Context package table
2. Exclusion list with reasons
3. Refresh trigger list
4. Top 3 context risks reduced

## Evaluation Rubric

A strong submission should:

- include only context that changes outcomes
- exclude clear noise and stale inputs
- define source precedence for conflicts
- define refresh triggers tied to workflow events
- assign context ownership explicitly

## Debrief Questions

1. Which context source caused the most risk before this exercise?
2. Which exclusion improved clarity the most?
3. Which trigger will force the most useful context refresh?
4. What context risk remains unresolved?
