# Module 05 Facilitator Notes

## Teaching Emphasis

- Teach context architecture as a system design task.
- Enforce include/exclude discipline.
- Require explicit refresh triggers.
- Use both anchors for transferability.

## Core Message

Reliable execution needs the right context package at the right time,
not maximum context volume.

## Common Student Mistakes

- treating all available information as useful context
- missing source precedence rules
- ignoring stale context risk
- no owner for context refresh actions
- no validation of context quality before execution

## Facilitation Prompts

- "What decision depends on this context item?"
- "What should be excluded and why?"
- "What event should force context refresh?"
- "Who owns context quality at this phase?"

## Suggested Timing

- 10 min: definitions and framing
- 15 min: context package and quality checks
- 20 min: dual-anchor exercise
- 10 min: debrief and failure-pattern review

## Exit Check

Before closing, ask each learner to provide:

- one context package for coding anchor
- one context package for knowledge-work anchor
- one refresh trigger they will add to real workflow
