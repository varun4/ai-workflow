# Module 01: Introduction

## Learning Objectives

- Understand what an AI workflow is
- Distinguish prompting from workflow design
- Recognize why agentic systems fail without structure

## Core Concept

AI agents become useful when they operate inside a clear workflow with
artifacts, boundaries, and evaluation. The course is about building
that workflow, not chasing better one-off prompts.

## Key Ideas

- Agent capability is only part of the system
- Workflow design determines reliability
- Artifacts such as specs and progress logs create continuity
- Evaluation and security are mandatory, not optional

## Reflection Questions

1. Where have you seen AI output fail because the request was underspecified?
2. Which part of your current workflow is least visible or least repeatable?
3. What work should never happen without a review step?
