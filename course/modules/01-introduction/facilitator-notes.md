# Module 01 Facilitator Notes

## Teaching Emphasis

- Define each concept before discussing it
- Use anchor examples early and repeatedly
- Keep language operational and testable
- Reframe weak output as workflow design failure

## Definitions To Enforce

Learners should be able to state:

- AI workflow: sequence of tasks, artifacts, decisions, checks
- ad hoc prompting: one-off requests without workflow controls
- executable request: bounded, testable, reviewable instruction set

## Common Student Mistakes

- Focusing on tools before defining the workflow
- Asking for automation before defining boundaries
- Treating prompts as a substitute for process design
- speaking in abstractions ("better", "improve", "high quality")
- skipping scope boundaries
- defining intent but not completion criteria
- missing approval boundaries for risky steps

## Facilitation Prompts

- "How would a reviewer know this is done?"
- "What would an agent need to guess here?"
- "What should be out of scope?"
- "Where must human approval be required?"

## Suggested Timing

- 10 min: framing and definitions
- 10 min: coding anchor walkthrough
- 10 min: knowledge-work anchor walkthrough
- 20 min: exercise
- 10 min: debrief

## Exit Check

Before closing, ask each learner to:

- define AI workflow in one sentence
- identify one ambiguity removed in each rewritten request
