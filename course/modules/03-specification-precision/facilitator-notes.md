# Module 03 Facilitator Notes

## Teaching Emphasis

- Define terms before discussing tactics.
- Keep examples anchored in real workflows.
- Treat precision as boundary design, not word count.
- Connect every spec field to validation quality.

## Core Message

A precise spec is an execution and evaluation contract.

## Definitions To Enforce

Learners should be able to state:

- specification precision: bounded and testable task definition
- executable spec: clear enough to execute without guessing
- non-goals: explicit out-of-scope boundaries

## Common Student Mistakes

- writing intent but not acceptance criteria
- skipping non-goals
- over-constraining low-risk details
- using vague quality terms without criteria
- missing approval boundaries

## Facilitation Prompts

- "How would a reviewer know this is complete?"
- "What would the agent still have to guess?"
- "Which risks are prevented by this non-goals section?"
- "What is strict here, and what is flexible?"

## Suggested Timing

- 10 min: definitions and framing
- 15 min: precision framework walkthrough
- 20 min: two-anchor rewrite exercise
- 10 min: debrief and pattern extraction

## Exit Check

Before closing, ask each learner to provide:

- one executable coding spec section set
- one executable knowledge-work spec section set
- one spec failure pattern they will prevent in their workflow
