# Module 04 Exercise

## Scenario

Decompose both anchor goals:

1. Coding: add role-based access checks to admin endpoints.
2. Knowledge work: prepare a vendor selection recommendation for
   leadership.

## Task

For each anchor, produce a decomposition map with:

- goal
- 3 to 5 phases
- tasks within each phase
- one concrete output per task
- checkpoint after each phase
- approval boundary where required
- validation method per phase

## Required Output Format

For each anchor, submit:

1. Phase map
2. Task list with outputs
3. Checkpoint decisions
4. Top 3 decomposition risks reduced

## Evaluation Rubric

A strong submission should:

- keep tasks bounded and testable
- assign one observable output per task
- include clear checkpoint and escalation points
- separate execution tasks from approval tasks
- avoid phases that are too broad to review

## Debrief Questions

1. Which phase was hardest to bound?
2. Which checkpoint would catch the most expensive failure?
3. Where did you choose fixed vs adaptive decomposition?
4. What is your biggest remaining decomposition risk?
