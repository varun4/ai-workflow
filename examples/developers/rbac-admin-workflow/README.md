# Developer Workflow Example: RBAC Admin Endpoints

## Scenario

Add role-based access checks to admin endpoints without breaking
existing API behavior.

## Purpose

This example shows a complete artifact-driven workflow for software
delivery using:

- `AGENTS.md`
- `SPEC.md`
- `PROGRESS.md`
- `SKILL_SECURITY_REVIEW.md`
- `EVAL_CHECKLIST.md`
- `APPROVAL_BOUNDARIES.md`

## Artifact Flow

1. Review operating rules in `AGENTS.md`.
2. Execute to the scope and constraints in `SPEC.md`.
3. Log each implementation phase in `PROGRESS.md`.
4. Apply reusable checks from `SKILL_SECURITY_REVIEW.md`.
5. Verify completion with `EVAL_CHECKLIST.md`.
6. Stop at `APPROVAL_BOUNDARIES.md` checkpoints.

## Learning Goal

Show how the same operating model used for course modules can be
applied to real feature delivery work.
