# Knowledge-Work Workflow Example: Vendor Selection Brief

## Scenario

Prepare a vendor selection recommendation for leadership using a
transparent, evidence-based workflow.

## Purpose

This example shows a complete artifact-driven workflow for knowledge
work using:

- `AGENTS.md`
- `SPEC.md`
- `PROGRESS.md`
- `SKILL_EVIDENCE_REVIEW.md`
- `EVAL_CHECKLIST.md`
- `APPROVAL_BOUNDARIES.md`

## Artifact Flow

1. Review operating rules in `AGENTS.md`.
2. Execute to the brief requirements in `SPEC.md`.
3. Log each phase in `PROGRESS.md`.
4. Apply reusable checks in `SKILL_EVIDENCE_REVIEW.md`.
5. Validate recommendation quality with `EVAL_CHECKLIST.md`.
6. Stop at `APPROVAL_BOUNDARIES.md` checkpoints.

## Module Deltas

- Module summary log: `MODULE_UPDATES.md`
- Detailed deltas:
  - `deltas/module-01.md`
  - `deltas/module-02.md`
  - `deltas/module-03.md`

## Learning Goal

Show that the same operating model used for engineering work also
supports high-stakes research and recommendation tasks.
